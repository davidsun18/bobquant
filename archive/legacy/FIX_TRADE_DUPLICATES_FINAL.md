## 问题重述

**用户反馈**:
> "交易记录里好多重复的记录，我看应该是没有成交的也记录在里边，以后只显示成交的记录，不要重复显示"

**实际问题**:
1. 主循环每 30 秒检查一次交易信号
2. 如果信号持续存在（如 MACD 金叉），会**重复执行买入**
3. 每次执行都扣款、加持仓、记录交易
4. 导致同一只股票在短时间内多次买入记录
   - 农业银行：9:25:23、9:27:00、9:28:37 各买入 4500 股（3 次）
   - 工商银行：类似情况

**错误理解**（第一次修复）:
- ❌ 限制每只股票每天只能买入 1 次
- ❌ 限制每只股票每天只能加仓 1 次

**正确理解**（本次修复）:
- ✅ **不要限制每天的买卖次数**
- ✅ **只有实际成交的才记录**
- ✅ **避免同一信号重复执行导致的重复成交**

---

## 修复方案

### 1️⃣ main.py - 避免重复信号执行

**新建仓检查**:
```python
# 检查本轮检查周期内是否已执行过买入
already_bought = any(t.get('code') == code and t.get('action') == '买入' for t in trades)
if already_bought:
    _log(f"  ⚪ {name}: 本轮已买入，跳过重复信号")
    continue
```

**加仓检查**:
```python
# 检查本轮检查周期内是否已执行过加仓
already_added = any(t.get('code') == code and '加仓' in t.get('action', '') for t in trades)
if already_added:
    _log(f"  ⚪ {name}: 本轮已加仓，跳过重复信号")
```

**关键点**:
- ✅ 检查的是**本轮检查周期**（trades 列表）
- ✅ **不是限制每天买卖次数**
- ✅ **而是避免同一信号重复执行**

### 2️⃣ web_ui.py - 从权威数据源读取

**修改前**:
```python
# 从交易记录文件读取（可能有重复）
trades = load_trades()  # 读取 sim_trading/交易记录.json
```

**修改后**:
```python
# 从账户 trade_history 读取（权威数据源）
account = load_account()
trades = account['trade_history'] if account else []
```

### 3️⃣ 清理工具

**cleanup_duplicates.py**:
- 清理现有重复记录
- 按 代码 + 时间 + 股数 + 价格 去重
- 备份原文件
