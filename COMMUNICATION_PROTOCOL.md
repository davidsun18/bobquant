  ## 🔗 通信链路                                                          
                                                                          
  ### 链路 1：数据 → 研究                                                 
```                                                                       
                                                                          
Data Bot 完成数据采集                                                     
   ↓                                                                      
发送通知：{"type": "data_ready", "data_type": "market", "date":           
"2024-01-15"}                                                             
   ↓                                                                      
Quant Research Bot 接收                                                   
   ↓                                                                      
开始因子计算                                                              
                                                                          
```                                                                       
                                                                          
  ### 链路 2：研究 → 开发                                                 
```                                                                       
                                                                          
Quant Research Bot 完成策略研发                                           
   ↓                                                                      
发送通知：{"type": "strategy_ready", "strategy_id": "MOM_001"}            
   ↓                                                                      
Dev Bot 接收                                                              
   ↓                                                                      
开始代码部署                                                              
                                                                          
```                                                                       
                                                                          
  ### 链路 3：开发 → 执行                                                 
```                                                                       
                                                                          
Dev Bot 完成代码部署                                                      
   ↓                                                                      
发送通知：{"type": "deployment_ready", "module": "trading_engine"}        
   ↓                                                                      
Execution Bot 接收                                                        
   ↓                                                                      
准备执行环境                                                              
                                                                          
```                                                                       
                                                                          
  ### 链路 4：执行 → 报告                                                 
```                                                                       
                                                                          
Execution Bot 执行完成                                                    
   ↓                                                                      
发送通知：{"type": "trade_complete", "trade_date": "2024-01-15"}          
   ↓                                                                      
Report Bot 接收                                                           
   ↓                                                                      
生成交易报告                                                              
                                                                          
```                                                                       
                                                                          
  ### 链路 5：合规监听                                                    
```                                                                       
                                                                          
Compliance Bot 监听所有交易消息                                           
   ↓                                                                      
实时风控检查                                                              
   ↓                                                                      
发现违规 → 立即拦截 + 告警                                                
   ↓                                                                      
正常 → 记录归档                                                           
                                                                          
```                                                                       
                                                                          
  ## 📨 消息格式                                                          
                                                                          
  ### 标准消息结构                                                        
  ```json                                                                 
  {                                                                       
    "message_id": "uuid",                                                 
    "timestamp": "2024-01-15T09:00:00+08:00",                             
    "sender": "data-bot",                                                 
    "receiver": "quant-research-bot",                                     
    "type": "notification",                                               
    "priority": "high",                                                   
    "payload": {                                                          
      "event": "data_ready",                                              
      "data": {...}                                                       
    }                                                                     
  }                                                                       
```                                                                       
                                                                          
### 消息类型                                                              
                                                                          
┌──────────────┬───────────────┬────────────┐                             
│ 类型         │ 说明          │ 处理方式   │                             
├──────────────┼───────────────┼────────────┤                             
│ notification │ 通知消息      │ 接收并处理 │                             
├──────────────┼───────────────┼────────────┤                             
│ request      │ 请求数据/服务 │ 响应请求   │                             
├──────────────┼───────────────┼────────────┤                             
│ response     │ 响应请求      │ 处理响应   │                             
├──────────────┼───────────────┼────────────┤                             
│ alert        │ 告警消息      │ 立即处理   │                             
├──────────────┼───────────────┼────────────┤                             
│ command      │ 控制命令      │ 执行命令   │                             
└──────────────┴───────────────┴────────────┘                             
                                                                          
🔐 安全规则                                                               
                                                                          
- 所有消息通过 Gateway 网关转发                                           
- 敏感数据加密传输                                                        
- 交易指令需要二次确认                                                    
- 合规消息优先处理                                                        
```
