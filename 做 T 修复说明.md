# 做 T 策略修复说明

**修复时间**: 2026-03-30 15:30  
**问题**: 建设银行做 T 卖出 3200 股后未接回，导致清仓  
**版本**: v2.2.19

---

## 🐛 问题诊断

### 现象
- 13:08-13:20 连续做 T 卖出 9 笔，共 3200 股
- 之后一股未接回
- 最终持仓：0 股 (清仓)
- **这不是做 T，是清仓卖飞！**

### 根本原因

#### ❌ Bug 1: 接回逻辑依赖持仓
```python
# 原代码
for code, pos in account.positions.items():
    # 检查做 T 接回
```

**问题**: 股票清仓后不在 `positions` 中，永远不会检查接回！

#### ❌ Bug 2: 接回门槛太高
```python
# 原配置
dip = 0.005  # 回落 0.5% 接回

# 建设银行实例
卖出价：¥9.53
接回价：需≤¥9.48 (回落 0.5%)
实际价格：可能一直在¥9.50-9.55，从未跌到¥9.48
```

#### ❌ Bug 3: 无兜底机制
- 收盘前没有强制接回
- 做 T 卖出的股票可能永久丢失

---

## ✅ 修复方案

### 修复 1: 做 T 接回独立于持仓

**修改文件**: `bobquant/main.py`

```python
# v2.2.19 新增：检查已清仓但做 T 未接回的股票 (兜底接回)
t_state = grid_t._state
for code, info in list(t_state.items()):
    if info.get('total_sold', 0) > 0 and not info.get('bought_back', False):
        # 有做 T 卖出但未接回，检查是否需要接回
        quote = quotes.get(code)
        if quote and quote['current'] > 0:
            bb_n, bb_reason = grid_t.check_buyback(code, quote['current'])
            if bb_n > 0:
                name = _find_name(s, code)
                t = executor.buy(code, name, bb_n, quote['current'], bb_reason, is_add=True)
                if t:
                    t['action'] = '🔄 做 T 接回 (兜底)'
                    trades.append(t)
                    grid_t.record_buyback(code)
```

**效果**: 即使股票清仓，也会检查并接回做 T 卖出的部分

---

### 修复 2: 降低接回门槛

**修改文件**: 
- `bobquant/strategy/engine.py`
- `bobquant/config/sim_config_v2_2.yaml`

```python
# 原配置
dip = 0.005  # 回落 0.5%

# 新配置
dip = 0.002  # 回落 0.2% (v2.2.19)
```

**效果对比**:

| 卖出价 | 原接回价 (0.5%) | 新接回价 (0.2%) | 差异 |
|--------|--------------|--------------|------|
| ¥9.53 | ¥9.48 | ¥9.51 | 更容易接回 |
| ¥100.00 | ¥99.50 | ¥99.80 | 更容易接回 |
| ¥255.00 | ¥253.73 | ¥254.49 | 更容易接回 |

### 修复 3: 收盘前强制接回

**修改文件**: `bobquant/main.py`

```python
# v2.2.19 新增：收盘前强制接回所有做 T 卖出的股票
from datetime import datetime
now = datetime.now()
time_str = now.strftime('%H:%M')

# 14:55 收盘前 5 分钟，强制接回
if '14:55' <= time_str <= '15:00':
    _log("  ⏰ 收盘前检查：强制接回所有做 T 卖出的股票")
    # 调用 force_buyback_all() 强制接回
```

**效果**: 避免做 T 卖出后忘记接回，确保日内平仓

## ⚠️ 注意事项

### 优点
- ✅ 做 T 卖出后会自动接回
- ✅ 降低接回门槛，更容易成交
- ✅ 收盘前强制平仓，避免隔夜风险

### 风险
- ⚠️ 接回门槛降低可能增加交易成本
- ⚠️ 强制接回可能在不利价格执行
- ⚠️ 频繁做 T 可能增加手续费

### 建议
- 监控手续费占比 (应<总盈利 10%)
- 观察接回成功率
- 根据实际效果调整接回门槛 (0.2-0.5%)
