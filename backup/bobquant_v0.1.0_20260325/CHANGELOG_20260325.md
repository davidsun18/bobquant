## 一、Bug 修复（3项）

### 1.1 交易记录不同步
- **问题**：交易引擎执行买卖后，只写入 `account_ideal.json` 的 `trade_history`，没有同步到独立的 `交易记录.json` 文件
- **影响**：Web UI 读取 `交易记录.json` 显示为空，用户看不到新交易
- **修复**：在每次交易执行后追加写入 `交易记录.json`
- **文件**：`ideal_sim_trading.py` → `bobquant/core/executor.py`

### 1.2 T+1 可交易数量计算错误
- **问题**：加仓场景下，`buy_date` 只记录首次建仓日期。今天加仓100股，但可交易数量仍显示全部股数
- **影响**：网页显示的可交易数量不正确，可能导致违规卖出
- **修复**：从交易记录中按日期统计当天所有买入笔数；底层引入 `buy_lots` 数组逐笔记录
- **文件**：`web_ui.py`、`bobquant/core/account.py`

### 1.3 process_guard 不守护交易引擎
- **问题**：交易引擎 `ideal_sim_trading.py` 需要手动启动，没有纳入进程守护
- **修复**：在 `process_guard.py` 中新增交易引擎守护项，支持交易时段自动启停
- **文件**：`process_guard.py`

## 二、交易引擎功能升级

### 2.1 三阶段架构（v2 → v3）

交易引擎从"扫描信号→买/卖"的单阶段，升级为三阶段顺序执行：

```
Phase 1: 做T（日内高抛低吸）
    ↓
Phase 2: 风控（止损 + 跟踪止损 + 分批止盈）
    ↓
Phase 3: 策略信号（MACD/布林带 + RSI过滤 + 量确认 → 建仓/加仓/减仓）
```

### 2.2 金字塔加仓
- **旧**：已持仓 → 跳过买入，无法加仓
- **新**：三档金字塔 3% → 5% → 7%
  - 首次建仓3%（轻仓试探）
  - 较均价跌≥3%时加仓到5%
  - 再跌≥3%加仓到7%（最高档）
  - 强信号（放量金叉）直接5%建仓

### 2.3 分批减仓
- **旧**：卖出信号 → 全部清仓
- **新**：策略减仓只卖50%可卖部分；RSI超买强信号卖70%

### 2.4 网格做T（v3优化）
- **旧（v2）**：固定阈值——日内涨3%高抛一次，每天限1次
- **新（v3）**：多档网格
  - 日内涨2%触发第一档
  - 每再涨1.5%触发下一档
  - 单日最多3次网格卖出
  - 每次卖出可卖部分的20%
  - 较最后一次卖出价回落≥1%自动接回

### 2.5 跟踪止损（新增）
- **旧**：只有固定止损（亏8%）和固定档位止盈
- **新**：盈利≥5%后激活跟踪止损
  - 持续跟踪记录最高价
  - 从最高价回撤≥2%自动卖出
  - 解决"涨10%又跌回来"的问题

### 2.6 RSI 过滤（新增）
- 买入信号需 RSI < 35（避免追高买入）
- RSI > 70 时卖出信号升级为强信号（加大减仓比例）
- RSI 过高的买入信号直接过滤，不执行

### 2.7 成交量确认（新增）
- 放量金叉（量比 > 1.5倍20日均量）标记为强信号
- 缩量信号（量比 < 0.8）视为弱信号，过滤掉
- 强信号建仓直接5%仓位（跳过3%试探档）

## 三、架构重构（核心改动）

### 3.1 旧架构
```
quant_strategies/
├── ideal_sim_trading.py    ← 974行，所有逻辑塞在一个文件
├── stock_pool_50.py        ← 硬编码股票池
├── itick_config.py         ← 散落的配置
├── web_ui.py
└── process_guard.py
```

**问题**：
- 32处硬编码路径（`/home/openclaw/...`）
- 数据获取、指标计算、策略判断、交易执行、风控、通知全部耦合
- 无法独立测试任何一个组件
- 换数据源/换券商需要大面积改代码

### 3.2 新架构
```
quant_strategies/
├── bobquant/                       # 新架构（1645行代码）
│   ├── __init__.py                 # 包声明 v0.1.0
│   ├── config/
│   │   ├── __init__.py             # 配置中心（187行）
│   │   ├── settings.yaml           # 全局配置（101行）
│   │   └── stock_pool.yaml         # 股票池（169行）
│   ├── indicator/
│   │   ├── __init__.py             # 导出接口
│   │   └── technical.py            # MACD/布林带/RSI/量比（61行）
│   ├── data/
│   │   └── provider.py             # 数据源抽象+腾讯实现（108行）
│   ├── core/
│   │   ├── account.py              # 账户+持仓+T+1（101行）
│   │   └── executor.py             # 买入/卖出/记录同步（157行）
│   ├── strategy/
│   │   └── engine.py               # MACD/布林带/风控/做T策略（245行）
│   ├── broker/
│   │   └── base.py                 # 券商抽象+模拟+实盘预留（154行）
│   ├── notify/
│   │   └── feishu.py               # 飞书通知（17行）
│   ├── web/                        # Web UI（待迁移）
│   ├── tests/
│   │   └── test_all.py             # 59个单元测试（355行）
│   ├── main.py                     # 三阶段引擎+主循环（239行）
│   └── logs/
├── run.py                          # 新入口
├── ideal_sim_trading.py            # 旧引擎（保留运行，过渡期）
├── web_ui.py                       # 旧 Web UI
└── process_guard.py                # 进程守护
```

### 3.3 设计原则

| 原则 | 实现 |
|------|------|
| **零硬编码** | 所有路径基于 `__file__` 自动推导，搬机器不改代码 |
| **配置驱动** | 所有参数集中在 `settings.yaml`，改参数不碰代码 |
| **可插拔** | 数据源、策略、券商都是抽象接口+工厂模式 |
| **向后兼容** | `to_legacy_config()` 生成旧版 CONFIG 字典 |
| **可测试** | 每个模块独立可测，59个单元测试覆盖全部核心逻辑 |

### 3.4 扩展方式

| 要做的事 | 怎么做 |
|----------|--------|
| 加新策略 | 继承 `BaseStrategy`，实现 `check()` 方法，注册到 `_strategy_map` |
| 加新数据源 | 继承 `DataProvider`，实现 `get_quote()`/`get_history()`，注册到工厂 |
| 加新指标 | 在 `indicator/technical.py` 中添加函数 |
| 对接实盘 | 继承 `BaseBroker`，实现买/卖/查持仓/查余额，`settings.yaml` 改 `broker.mode` |
| 改交易参数 | 只改 `settings.yaml`，不碰代码 |

## 六、配置文件说明

`bobquant/config/settings.yaml` 是系统唯一的配置文件，修改参数只需改这个文件：

```yaml
# 改初始资金
account.initial_capital: 1000000

# 改止损线（从-8%改为-5%）
strategy.stop_loss_pct: -0.05

# 关闭做T
strategy.grid_t.grid_max: 0

# 切换数据源
data.primary: "infoway"

# 未来切实盘
broker.mode: "easytrader"
```
