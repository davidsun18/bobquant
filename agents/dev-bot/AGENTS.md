```markdown                                                               
  # AGENTS.md - Dev Bot 行为准则                                          
                                                                          
  ## 💻 核心原则                                                          
                                                                          
  **"没有测试的代码是技术债，没有文档的代码是黑盒"**                      
                                                                          
  ### 开发流程                                                            
  1. 需求理解 → 2. 设计评审 → 3. 代码实现 → 4. 单元测试 → 5. 集成测试 → 6.
代码审查 → 7. 部署上线                                                    
                                                                          
  ### 代码规范                                                            
                                                                          
  **Python 规范：**                                                       
  - PEP8 代码风格                                                         
  - 类型注解 (Type Hints)                                                 
  - 函数文档字符串 (Docstring)                                            
  - 异常处理 (try/except)                                                 
                                                                          
  **C++ 规范：**                                                          
  - C++ Core Guidelines                                                   
  - RAII 资源管理                                                         
  - 智能指针 (unique_ptr/shared_ptr)                                      
  - const 正确性                                                          
                                                                          
  ### 测试要求                                                            
                                                                          
  **单元测试覆盖率：**                                                    
  | 模块类型 | 最低覆盖率 |                                               
  |---------|-----------|                                                 
  | 核心交易逻辑 | 95% |                                                  
  | 风控模块 | 95% |                                                      
  | 数据处理 | 85% |                                                      
  | 工具函数 | 80% |                                                      
                                                                          
  **测试用例必须覆盖：**                                                  
  - [ ] 正常路径                                                          
  - [ ] 边界条件                                                          
  - [ ] 异常输入                                                          
  - [ ] 并发场景                                                          
                                                                          
  ### 代码审查清单                                                        
  - [ ] 代码通过编译/解释                                                 
  - [ ] 所有测试通过                                                      
  - [ ] 无安全漏洞                                                        
  - [ ] 无性能瓶颈                                                        
  - [ ] 日志完备                                                          
  - [ ] 错误处理完善                                                      
  - [ ] 文档齐全                                                          
```
