---

## 📊 冲突场景

### 测试场景
```
持仓：成本¥10.0 × 1000 股
开盘：¥10.2
现价：¥10.6

日内涨幅：(10.6-10.2)/10.2 = +3.9%
总盈利：(10.6-10.0)/10.0 = +6.0%
```

### 触发结果

**做 T 规则**:
- 条件：日内涨幅 ≥ 2% ✅ (3.9% ≥ 2%)
- 操作：卖出 20% = 200 股

**止盈规则**:
- 条件：总盈利 ≥ 5% ✅ (6.0% ≥ 5%)
- 操作：L1 止盈 33% = 330 股

**问题**: 同一时间触发两次卖出！重复交易！

---

## 🔧 修复方案

### 优先级控制

**原则**: 风控优先于做 T

```python
# Phase 2 风控 (止盈/止损) - 高优先级
# Phase 1 做 T - 低优先级

# 修复逻辑:
1. Phase 1 开始时，先检查所有持仓的风控状态
2. 已触发止盈/止损的股票，标记为 risk_checked
3. 做 T 检查时，跳过 risk_checked 的股票
```

### 代码实现

```python
# v1.1.3 修复：先检查哪些股票触发了止盈/止损
risk_checked = set()
for code, pos in account.positions.items():
    result = risk.check(code, pos, current_price)
    if result['action']:
        risk_checked.add(code)  # 标记为风控优先

# 做 T 时跳过已触发风控的股票
for code, pos in account.positions.items():
    if code in risk_checked:
        continue  # 跳过，避免冲突
    # 执行做 T 检查...
```

## 📋 执行顺序

### Phase 1.5: 风控预检查 (新增)
```
在 Phase 1 做 T 之前:
1. 遍历所有持仓
2. 检查是否触发止盈/止损
3. 标记已触发风控的股票
```

### Phase 1: 网格做 T (优化)
```
遍历所有持仓:
1. 检查是否在 risk_checked 集合
2. 如果是 → 跳过 (避免冲突)
3. 如果否 → 执行做 T 检查
```

### Phase 2: 风控 (正常执行)
```
遍历所有持仓:
1. 检查止盈/止损/跟踪止损
2. 执行风控操作
```

## 🎯 优先级规则

| 规则 | 优先级 | 说明 |
|------|--------|------|
| **止损** | 🔴 最高 | 亏 8% 立即卖出 |
| **止盈** | 🔴 高 | 盈利 5%/10%/15% 分批卖出 |
| **跟踪止损** | 🟡 中 | 盈利后回撤 2% 卖出 |
| **做 T** | 🟢 低 | 日内波动高抛低吸 |

**原则**: 高优先级规则触发时，低优先级规则跳过
